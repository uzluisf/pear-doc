Config

  • directories

This is where you specify the directories Pear must be aware of. They're a a content directory, a templates directory, an include directory, and an output directory.

directories:
    content: content
    templates: templates
    include: static
    output: public

Here, Pear will look for content under content, for templates under templates, for static files under static. It will place the generated HTML under the public directory.

  • date-format

The format that will be used for a post/page unless it already includes a formatting of its own.

Excluding directories, everything else (including date-format) will be made available to templates in the global variables site.
